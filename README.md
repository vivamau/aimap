# Atlas of Artificial Intelligence

A cartographic survey of AI models worldwide, with a Node.js editorial backend that persists changes to a JSON store and regenerates a public GeoJSON file consumed by the static viewer.

```
.
├── index.html                      # Public viewer (static — GitHub-Pages friendly)
├── admin.html                      # Editorial Desk UI (talks to the backend)
├── assets/
│   ├── css/{atlas,desk}.css        # Editorial cartographic theme
│   └── js/{atlas,desk}.js          # D3 map + admin form
├── data/
│   └── ai-models.geojson           # ◀ generated by the backend; read by index.html
└── backend/
    ├── data/ai-models.json         # Source of truth (the JSON "database")
    ├── services/                   # db, geojsonExporter
    ├── routes/                     # models, meta, export
    ├── middleware/                 # errorHandler
    ├── tests/                      # Jest + Supertest (42 tests, ≥85% coverage)
    ├── swagger.json                # OpenAPI 3 spec served at /api/docs
    ├── app.js / index.js           # Express app + bootstrap
    └── scripts/export-geojson.js   # Standalone exporter (CI / pre-commit)
```

## Run

```bash
cd backend
npm install
npm start            # → http://localhost:3000
npm test             # 42 Jest + Supertest specs
npm run test:coverage
```

When the server boots it:

1. Loads `backend/data/ai-models.json` into memory.
2. Regenerates `data/ai-models.geojson` (atomic write).
3. Serves the static site (`/`, `/admin.html`, `/assets`, `/data`) and the API.

## Editing flow

1. `npm start`, open <http://localhost:3000/admin.html>.
2. Edit, add or remove entries — every save hits the API and regenerates the GeoJSON.
3. Commit both `backend/data/ai-models.json` and `data/ai-models.geojson` to publish.

## Publishing the public viewer

The viewer is purely static. For GitHub Pages you only need:

- `index.html`
- `assets/css/atlas.css`, `assets/js/atlas.js`
- `data/ai-models.geojson`

The `backend/` and `admin.html` are admin-only and should be excluded from the public deployment.

## API

All endpoints under `/api`. Full OpenAPI at <http://localhost:3000/api/docs>.

| Method | Path                  | Description                                            |
|--------|-----------------------|--------------------------------------------------------|
| GET    | `/api/health`         | Liveness probe                                         |
| GET    | `/api/meta`           | Get atlas meta                                         |
| PUT    | `/api/meta`           | Update edition / compiler                              |
| GET    | `/api/models`         | List all models                                        |
| GET    | `/api/models/:id`     | Get one                                                |
| POST   | `/api/models`         | Create (auto-derives id from name) + regenerate GeoJSON|
| PUT    | `/api/models/:id`     | Update + regenerate GeoJSON                            |
| DELETE | `/api/models/:id`     | Delete + regenerate GeoJSON                            |
| POST   | `/api/export`         | Force a regeneration (no DB change)                    |

## GeoJSON shape

Standard `FeatureCollection`. Each feature is a `Point` with `[lng, lat]`; all model fields live under `properties`. A non-standard `meta` field at the top carries edition / updated / compiler.
